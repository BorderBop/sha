# sha

Аркадная игра про захват территории (в духе Qix): курсор тянет за собой линию от рамки/уже захваченных блоков, отрезанные области закрашиваются, а бегающие по полю шарики норовят задеть недорисованную линию и сорвать попытку.

Проект в первую очередь демонстрирует связку двух библиотек:

- **[pygame-ce](https://pyga.me/)** — вся игровая логика, физика и отрисовка написаны на нём как обычная desktop-игра.
- **[pygbag](https://github.com/pygame-web/pygbag)** — компилирует ту же самую pygame-игру в WASM и запускает прямо в браузере, без переписывания под JS. Один и тот же код (`main.py` + модули) работает и как `python3 main.py` на компьютере, и как полноценная веб-игра.

Поверх этого — небольшой сервер на чистом Python (`leaderboard_server.py`, без фреймворков, только stdlib + SQLite) с простой авторизацией по имени и 4-значному PIN и глобальной таблицей рекордов, к которой браузерный клиент обращается через `pygbag`'ный HTTP-fetch.

## Возможности игры

- Захват территории линией курсора с проверкой пересечения через flood-fill по сетке
- Шарики-помехи: с каждым уровнем добавляется ещё один
- Уровни (переход при захвате заданного % поля), 3 жизни, экран Game Over с рестартом
- Подсчёт очков (процент × сложность − штраф за время), несгораемая сумма между уровнями
- Пауза (Space), боковая информационная панель (уровень / прогресс / очки / жизни / таблица рекордов)
- Логин по имени + PIN, глобальная таблица лучших результатов

## Структура проекта

| Файл | Назначение |
|---|---|
| `main.py` | точка входа, игровой цикл, экран логина, HUD |
| `config.py` | все константы (размеры, цвета, скорости, адрес API) |
| `ball.py`, `cursor.py`, `obstacles.py` | игровые сущности |
| `physics.py` | столкновения шариков друг с другом и с препятствиями |
| `trail.py` | логика линии курсора: касание стен, обрыв от шарика |
| `capture.py` | flood-fill захват отрезанной территории |
| `scoring.py` | формула подсчёта очков |
| `images.py` | загрузка спрайтов |
| `net.py` | клиентская сеть — единственное место, где используется `pygbag`-специфичный `aio.fetch` (в браузере) |
| `leaderboard_server.py` | сервер таблицы рекордов: HTTP + SQLite, без внешних зависимостей |
| `pics/` | спрайты |

## Запуск локально (разработка)

Обычный desktop-режим, без браузера и без сервера очков:

```bash
python3 main.py
```

Полный стенд (игра в браузере + сервер очков) одной командой:

```bash
./run.sh
```

Поднимет `leaderboard_server.py` на `http://localhost:8765` и pygbag dev-сервер на `http://localhost:8000`. `Ctrl+C` останавливает оба.

## Деплой на сервер (Hetzner)

Игра развёрнута на `games.glamkit.ai`. Сервер очков живёт в Docker-контейнере, статика игры и HTTPS отдаются уже существующим на сервере системным Caddy (не отдельным контейнером — поэтому в `docker-compose.yml` нет своего Caddy, только `leaderboard`).

### Первоначальная настройка

```bash
cd /opt
git clone https://github.com/BorderBop/sha.git
cd sha
```

В `config.py` для продакшн-сборки поменять:
```python
LEADERBOARD_BASE_URL = "/api"   # вместо "http://localhost:8765"
```
(относительный путь — резолвится браузером в тот же домен, откуда отдана страница; для локальной разработки эта строка должна оставаться `http://localhost:8765`, там нет прокси перед сервером очков)

Собрать статику и поднять контейнер сервера очков:
```bash
python3 -m pygbag --build main.py
docker compose up -d --build
```

Добавить блок из `Caddyfile` этого репозитория в конец существующего `/etc/caddy/Caddyfile` (путь в `root` должен указывать на `/opt/sha/build/web`), затем:
```bash
caddy validate --config /etc/caddy/Caddyfile
systemctl reload caddy
```

### Обновление (после изменений в коде)

```bash
cd /opt/sha
git pull

# если менялся клиент (main.py и другие игровые модули)
python3 -m pygbag --build main.py

# если менялся leaderboard_server.py
docker compose up -d --build

# Caddyfile обычно трогать не нужно повторно
```

Таблица рекордов хранится в именованном Docker-volume (`leaderboard_data`) и **не теряется** при обычном обновлении — volume переживает пересборку и перезапуск контейнера. Стирается только явным `docker compose down -v`.

Проверка после деплоя:
```bash
curl https://games.glamkit.ai/api/ping
```
